Understanding the JS Event Loop

The event loop coordinates the call stack and the callback queue. Synchronous code runs first; async callbacks (like setTimeout, fetch responses, and microtasks) are processed when the stack is clear.

  • Microtasks (Promises, queueMicrotask) run before macrotasks.
  • Long tasks block rendering; keep them small or offload to workers.

Closures in real life

A closure is just a function remembering the scope in which it was created. Great for encapsulation, factories, and private state.

Tip: Avoid accidentally closing over loop variables; prefer let for block scoping.

Amir Ahmed • 2h ago

Promises vs async/await

They use the same underlying model. Async/await is syntactic sugar that makes sequential flows easier to read, while Promise combinators (all, allSettled, any, race) excel at concurrency.

Hannah Scott • 5h ago

Hoisting: what actually happens

Function declarations are hoisted with their definitions. var declarations are hoisted and initialized to undefined; let and const are hoisted but remain in the temporal dead zone until initialized.

Ava Martin • 1d ago

map vs forEach vs reduce

  • map: transforms to a new array of the same length.
  • forEach: iterates for side-effects; returns undefined.
  • reduce: accumulates to a single value (can also build arrays/objects).
